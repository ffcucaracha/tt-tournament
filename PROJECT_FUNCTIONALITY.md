# Функционал проекта (актуально)

## Что это за система
Приложение для проведения любительских турниров по настольному теннису в клубном формате (обычно 10–30 участников, турнир может идти один или несколько дней).

Сейчас поддерживается **только швейцарская система (Swiss)**.

## Для кого
- **Администратор**: настраивает турнир, добавляет/импортирует участников, запускает туры, вносит результаты.
- **Публичные пользователи (игроки/зрители)**: без авторизации смотрят участников, текущий тур, историю туров, расписание и таблицу.

## Основные возможности
- Создание и управление турнирами.
- Поддержка одного активного турнира в момент времени.
- Регистрация участников вручную и импортом CSV.
- Посев/запуск турнира из статуса `draft`.
- Генерация туров по Swiss-правилам.
- Ввод результатов матчей.
- Автоматический учет `bye` (при нечетном числе участников).
- Подсчет таблицы: очки, победы/поражения, `bye`, Buchholz.
- Публичные экраны для просмотра хода турнира.
- TV-режим для вывода на общий экран.

## Как работает турнир (Swiss)
- Участник **не выбывает** после поражения.
- Турнир идет по турам (`roundsCount` задается в настройках).
- Организатор может остановиться после завершённого тура и продолжить позже.
- За победу: **1 очко**.
- За `bye`: **1 очко**.
- `bye` не считается соперником в Buchholz.
- Следующие пары формируются по текущим результатам и истории встреч.
- Повторные пары избегаются, а если это невозможно — используются как явный fallback.
- Таблица сортируется по:
  1. очкам,
  2. Buchholz,
  3. техническому стабильному критерию.

## Что видно в интерфейсе
### Публичная часть
- Список участников.
- Текущий тур.
- История туров.
- Расписание матчей.
- Таблица результатов (включая Buchholz).

### Админка
- Турниры (создание, активация, завершение).
- Участники (добавление, редактирование, удаление в `draft`, CSV-импорт).
- Матчи и ввод результатов.
- Генерация следующего тура.

## Ограничения и правила состояния
- Участников можно менять только в `draft`.
- Турнир нельзя запустить с менее чем 2 участниками.
- Следующий тур нельзя сгенерировать, пока текущий не завершен.
- После завершения турнира результаты не должны редактироваться обычным потоком.

## Техническая основа
- **Backend**: NestJS + TypeScript + Prisma + PostgreSQL.
- **Frontend**: React + TypeScript + Vite + TanStack Query + Tailwind.
- **Инфраструктура**: Docker Compose.

## Источники данных участников
Поддерживается импорт из CSV (частый сценарий: данные из Google Forms/Яндекс Форм после выгрузки).

## Локальный запуск
- Dev-контур поднимается через `docker-compose.dev.yml`.
- Основные сервисы: `backend-dev`, `frontend-dev`, `postgres`.
